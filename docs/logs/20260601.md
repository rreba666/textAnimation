## 技术决策记录

### 天气 API：Open-Meteo（免费，无需 Key）
- 两步调用：地理编码 API → 天气预报 API
- WMO 代码映射中文描述 + lucide-react 图标
- 10 分钟缓存

### 主题切换：CSS 变量 + documentElement.setAttribute
- TailwindCSS 所有颜色引用 CSS 变量，双主题无需 `dark:` 前缀
- `toggleTheme()` 即时切换 `data-theme` 属性
- body / card 保留 CSS transition 作为日常过渡

### 状态管理：Zustand v5 persist
- 单 store 管理全部模块（todos / habits / notes / links / weather / theme）
- persist 中间件自动同步 localStorage
- HMR 保护：`import.meta.hot.accept` 中转移旧 store 状态到新 store

### 状态响应式踩坑记录
1. **Habits.tsx 订阅函数引用**：`useDashboardStore((s) => s.getWeekRecords)` 返回稳定函数引用，不触发重渲染。修复：订阅 `habitRecords` 数据源。
2. **HydrationGate 误用**：`hasHydrated()` 非响应式调用，且 localStorage 水合同步完成，HydrationGate 无意义。已移除。
3. **version 字段**：persist 中声明 `version` 但无 `migrate` 时，Zustand v5 可能丢弃数据。已移除。
4. **StrictMode**：dev 双次渲染干扰外部 store。已移除。

### 主题动画迭代
1. 第一版：View Transitions API（兼容性差）
2. 第二版：Element Plus 风格圆形扩散（时序难以控制，闪烁）
3. 最终版：直接切换 + CSS transition（简洁可靠）
